# Soul-Steel

Steel-city action-RPG. Planning lives in markdown scattered across the repo (this is the *before*
state — no framework yet). Notes for whoever picks this up:

- **Decisions** are ADR-style under `docs/decisions/`, numbered `NNNN-…`. Header says `Status:`.
- **Retros / spikes** land in `reports/`, dated `YYYY-MM-DD-…`, marked draft or complete.
- **The delivery board** tracks workstreams in `delivery/`. Columns we move cards through:
  **design → qualify → build → ship**. Parked work sits in its own pile.
- **Roadmap** is `docs/roadmap.md`.
- **Work logs** go in `journal/`, one dated file per working session.
